# Token Harbor 長線內容藍圖

> 狀態：產品規劃草案，只作方向記錄，今次不代表已排期或開始實作。
> 範圍：只討論單人與本地長線內容；排行榜、好友港口及其他社交功能另行規劃。

## 1. 核心判斷

三個港口、更多魚和更高數字，只能延後內容耗盡，不能真正解決兩三個月後的沉悶。

長線留存需要同時提供四種新鮮感：

1. **新地方**：新群島、魚種、船和視覺氣候。
2. **新規則**：每片海域改變玩家的判斷與操作，而非只提高生命值。
3. **新目標**：收藏、研究、故事、專精與長期建設。
4. **新變化**：同一海域會因潮汐、遷徙、事件和季節產生不同局面。

Token Harbor 的定位仍然是 Codex 工作陪伴遊戲。它應該讓等待時間有趣，但不能要求玩家為了遊戲延遲工作。

## 2. 不可破壞的產品原則

- 手動捕魚仍是主要遊戲；自動船只負責降低重複勞動。
- Sail Power 仍然主要來自真實 Codex Token 使用，不能被普通掛機完全取代。
- 一次手動內容以約 60–120 秒為主，可中途返航，可被 Codex 工作狀態暫停。
- 離開數星期再回來不會受懲罰，不設強制連續簽到。
- 新島必須帶來至少一個新玩法規則，不能只是換背景和提高數字。
- 長線養成以橫向選擇配合縱向升級，避免只有唯一最強配置。
- 不靠社交壓力、付費抽卡或限時焦慮維持活躍；可控機率與收藏玩法另見機率玩法草案。
- 全球版本所有系統必須能本地化，固定面板內的文字要保持短而清楚。

## 3. 六個常駐內容支柱

### A. 分支遠征圖

普通航線保持短局；另設可選的「遠征」模式，讓玩家用一次較大的 Sail Power 投入，進行 3–6 個節點的航程。

每個節點可能是：

- 魚群：手動捕魚。
- 海盜：戰鬥或繞路。
- 沉船：打撈物資，但增加船體風險。
- 洋流：縮短航程或把船帶往未知節點。
- 補給點：修理、冷藏或更換裝備。
- 異常天氣：選擇返航、避風或冒險前進。
- 傳說蹤跡：累積線索後進入特殊狩獵。

玩家每一步都可以帶現有漁獲返航。這個系統提供真正的風險判斷，而不是把 90 秒航程單純拉長。

### B. 船隻專精與出海配置

每艘新船都由小漁船開始，沿「小漁船 → 拖網船 → 遠洋船 → 深海船」逐級改裝；買新船不能直接跳到高級船型，設備亦由 LV.1 重新養成。玩家可以分開選擇「目前管理／升級的船」和「親自出海的主力船」，其餘船才可加入自動派遣。

四級船型是縱向成長骨架，但不應令所有終局船完全相同。未來可在指定船型或改裝階段加入用途專精：

- 快艇：航程短、追魚靈活、容量低。
- 拖網船：普通魚收益高，但稀有率較低。
- 研究船：可以標記、拍攝和採樣，出售收益較低。
- 裝甲漁船：對抗海盜和風暴，速度較慢。
- 冷藏船：容量大，適合遠征和大型訂單。
- 深潛母船：進入高壓海域，營運與維修成本高。

出海前配置只保留少量重要選擇，例如兩個設備槽、一種漁具和一名主船員。配置需要有取捨，避免演變成複雜背包管理。

### C. 海洋圖鑑與研究

捕到魚只是圖鑑的第一層。每個物種可以有四種長線記錄：

1. 首次發現。
2. 尺寸或品質紀錄。
3. 特殊行為觀察。
4. 完整生態研究。

研究任務不一定要求捕殺：可以拍攝、標記、護送、採集水樣或清除廢棄漁網。完成研究會解鎖魚類故事、海域情報、裝飾、船體塗裝或小量永久能力。

研究進度是收藏目標，不新增可大量囤積的貨幣。避免再加入一種需要刷取的「研究點數」。

### D. 船員職涯與故事

船員不只升數字，達到指定等級後選擇專精路線：

- 船長：探險、商務或風暴導航。
- 漁夫：速度、精準或大型魚處理。
- 工程師：省油、維修或設備超頻。
- 未來職業：生物學家、砲手、潛水員、廚師。

每名主要船員有短故事鏈和個人任務。選錯專精可以在港口重新訓練，不用永久廢掉角色，也不使用隨機抽取船員。

### E. 港口功能區與外觀成長

港口擴建分成有作用的區域，而不是單純樓層數字：

- 造船廠：船隻、修理和設備專精。
- 海洋研究所：圖鑑、標記與環境任務。
- 魚市場：訂單、拍賣與價格情報。
- 保育站：恢復海域、救援與保護任務。
- 燈塔區：新航道、天氣預報與遠征情報。
- 博物館／水族館：展示完整收藏和特殊紀念物。

建築同時改變港口主畫面的外觀。玩家長期投入後，第一眼便能看出自己的港口成長，而不是只看到數字。

### F. 傳說狩獵與大型事件

傳說內容應是技巧與準備測試，而非普通魚加十倍生命：

- 分階段追蹤蹤跡。
- 觀察特殊移動模式。
- 在限定船艙重量內決定帶走哪些漁獲。
- 處理海盜、暗礁或風暴干擾。
- 最後階段可以選擇捕獲、標記研究或護送保育。

完成後取得永久紀念物、港口展示、船體塗裝與故事，不把最高價值全部放在金幣上。

## 4. 讓舊海域持續變化的世界系統

### 潮汐與洋流

每個時段改變部分魚群、航程方向和風險。預報必須清楚，玩家不需要查攻略。

### 季節遷徙

魚種會在數星期的寬鬆窗口出現於不同海域。錯過的物種會定期回歸，不製造永久錯失。

### 海域狀態

例如水溫升高、浮游生物爆發、沉船漂流帶或海盜封鎖。每個狀態改變 1–2 條規則，維持可讀性。

### 事件牌組

每條航線擁有自己的事件牌組。新版本可以加入事件而不用重造整個海域，使舊航線逐步變豐富。

### 個人航海週誌

記錄首次發現、最大尺寸、危險返航、傳說事件和港口里程碑。只記遊戲事件，不讀取 Codex 對話、檔案內容或工作成果。

## 5. 群島章節藍圖

每個章節以三個主要港口組成。每個港口至少包含：一種海域規則、6–8 種新生物、一個威脅、一條故事鏈、一個港口功能與一場傳說事件。

### Chapter I — Home Waters

現有內容：

- Driftwood：基礎操作、近岸魚群與小型海盜。
- Coral Port：珊瑚海域、敏捷熱帶魚與容量管理。
- Mistlight：大霧、深海魚、燈塔與高風險航線。

作用：完成核心循環與第一套船隊。

### Chapter II — Broken Compass

- Emberwake：火山洋流、熱泉噴發、設備過熱。
- Mangrove Crown：潮汐水道、淺水船型、狹窄追逐。
- Frostline Haven：浮冰、低溫保鮮優勢、船體結冰。

作用：引入船隻專精、簡單出海配置與第一批分支遠征。

### Chapter III — Outer Belt

- Tempest Reach：持續風暴、雷擊風險、風暴導航。
- Sunken Crown：遺跡、沉船打撈、有限氧氣事件。
- Midnight Trench：高壓深潛、生物發光、聲納盲區。

作用：完整研究系統、船員專精與多階段傳說狩獵。

### Chapter IV — Open Ocean

不再以固定小島為主，而是移動航線、季節遷徙、臨時補給船和未知海圖。

作用：成為真正可長期擴充的終局區域；新事件、魚群和遠征節點可以持續加入，不需要無限堆島。

## 6. 非社交內容更新節奏

更新不使用每日強迫任務，改用較寬鬆的內容層級：

| 節奏 | 內容 | 玩家壓力 |
| --- | --- | --- |
| 每次出海 | 隨機事件、魚群行為、天氣與路線選擇 | 極低 |
| 每週世界變化 | 洋流、魚群遷徙、特殊訂單 | 錯過不受罰 |
| 每月小更新 | 新事件牌、研究任務、裝飾或一種魚 | 可永久回收 |
| 每季大更新 | 新港口、故事章節、船或核心玩法 | 永久內容 |

所有輪替內容應進入回歸池。限定外觀可以再次出現；重要能力不可永久絕版。

## 7. 建議開發順序

### Stage 1 — 先令現有三港耐玩

1. 海洋圖鑑第二層：尺寸、行為與研究紀錄。
2. 每條航線加入小型事件牌組。
3. 船隻專精與兩套可保存配置。
4. 港口外觀隨設施等級真正改變。
5. 第一個多階段傳說狩獵。

原因：這些系統能重新利用現有魚、船和三張背景，內容回報最高。

### Stage 2 — Chapter II

1. 新的可平移／縮放群島地圖。
2. Emberwake、Mangrove Crown、Frostline Haven。
3. 分支遠征的最小版本。
4. 船員第一次專精選擇。
5. 造船廠、研究所與保育站。

### Stage 3 — Chapter III 與終局

1. 完整遠征圖與打撈節點。
2. 多階段天氣和深潛規則。
3. 船員故事鏈。
4. 傳說狩獵系列。
5. Open Ocean 的可持續事件框架。

## 9. 經濟與貨幣限制

長線系統很容易被貨幣數量拖垮，因此暫定：

- Sail Power：來自 Codex 使用，負責出海與開港。
- Coins：魚獲、訂單與港口經濟。
- Materials：設備、船體與特殊建設；現有欄位可正式啟用。
- 研究、故事、聲望和圖鑑以進度表示，不新增可刷貨幣。

原則上同一時間不讓玩家管理超過三種可花費資源。

## 10. 明確不做

- 不做強制每日簽到、斷簽懲罰或必須準時上線的任務。
- 不做付費抽卡船員、裝備詞條洗鍊或付費戰力；核心船員與設備必須有確定取得路線。
- 不做每次擴張便清空港口的傳統轉生重置。
- 不用大量製作倒數計時器把遊戲變成收菜工具。
- 不讓自動船取代手動稀有魚、傳說狩獵和主要發現。
- 不用十多種材料、票券和活動幣製造假深度。
- 不把同一條魚單純換色、加血、加價後當成新內容。
- 不讓遊戲通知、事件或倒數阻擋 Codex 工作。

## 11. 技術方向記錄

為了讓未來增加港口不需要大改，後續實作應逐步改成資料驅動：

- 島嶼、港口、航線、魚類、威脅和事件使用穩定 ID 的內容註冊表。
- 地圖由港口資料生成，不再永久寫死三個標記。
- 每個新港口自動開放四個養成等級；全域上限可以隨章節提高。
- 遠征節點使用可版本化的事件定義，舊存檔只保存結果與進度。
- 所有新增存檔欄位必須有遷移與合理預設值。
- 輪替世界狀態可由本地種子和時間計算，離線單人版不依賴伺服器。
- 未安裝新內容包時，舊版本仍可正常讀取自己的存檔。

## 12. 驗收標準

長線內容不是以「有多少島」判斷，而以玩家是否仍有不同選擇判斷：

- 第 30 日：至少有兩套用途不同的有效出海配置。
- 第 60 日：舊航線仍會因研究、事件或世界狀態產生新目標。
- 第 90 日：玩家接觸到分支遠征或傳說狩獵，而不是只升更大的數字。
- 離開一個月後回歸：沒有斷簽損失，並能快速理解目前世界變化。
- 每個新港口：至少改變一條核心規則，不只是新增美術資源。
- 手動出海：仍然可以在 Codex 等待空檔內完成或安全暫停。

## 13. 目前決策

1. 三個現有港口定義為 Chapter I，而非完整世界。
2. 長線優先投資事件、研究、配置和遠征，不靠社交維持內容。
3. 新港口繼續每港開放四級養成，但不把單純升級當作主要新內容。
4. 不採用港口全部重置的轉生系統；使用新海域許可、專精與收藏延續成長。
5. Stage 1 應先提升現有資產的重玩價值，再大量製作新島。
6. 可控運氣、收藏包與保底方向記錄於 [機率與收藏長線玩法草案](randomized-progression-roadmap.md)，目前未排期、未實作。
